# Reference Catalog Provenance

The JSON files in this directory contain identity fields used to validate
foreground links: activity or flow names, reference products, locations,
categories, and units. They do not contain exchange amounts or complete
background inventories.

The ecoinvent catalogs were generated by
`scripts/generate_reference_catalogs.py` from locally installed, licensed
ecoinvent releases. The UVEK 2025 technosphere catalog was generated from a
locally supplied Brightway-compatible workbook; its biosphere identities came
from the ecoinvent 3.10 biosphere database.

`RESOURCE_MANIFEST.json` records exact SHA-256 hashes, sizes, profiles, and
identity counts. Its `legal_review_required` status is intentional: maintainers
must confirm that distribution of these derived identity catalogs is permitted
under the applicable source-data agreements before a stable public release.
The manifest is an integrity and provenance record, not a license grant.
